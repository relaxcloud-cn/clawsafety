# ClawSafe 竞品综合评估与策略

**更新时间**: 2026-03-23

---

## 一、竞品整体评级

### 排名：谁做得最好？

| 排名 | 产品 | 综合评价 | 核心理由 |
|:----:|------|---------|---------|
| 1 | **Snyk Agent Scan** | 最强 | 技术深度 + 品牌 + 生态整合，是当前赛道标杆 |
| 2 | **奇安信 SafeSkill** | 最全 | 唯一做到扫描 + Hub + 沙箱 + 企业方案全覆盖 |
| 3 | **Cisco Skill Scanner** | 最硬核 | CFG 数据流分析是纯技术层面最强的静态分析引擎 |
| 4 | **SkillSafe** | 最务实 | 双向验证模型简洁有效，产品化做得最好 |
| 5 | **Socket** | 最有数据 | 60K 规模扫描，但自身产品独立性弱 |

### 逐个点评

**Snyk Agent Scan — 标杆，但有商业锁定**

Snyk 是目前做得最好的。原因不是技术最强，而是**打法最聪明**：开源 CLI 圈开发者 → Web Inspector 降低门槛 → 后台模式卖给企业 → 与 Vercel/Tessl 合作嵌入生态。LLM Judge + 规则的混合引擎在深度和速度间取得了好的平衡。Toxic Flow 检测（分析 prompt 如何触发恶意行为）是真正有技术含量的创新。

弱点：核心能力绑定 Snyk Evo 平台，独立使用有限制。不做中文市场。

**奇安信 SafeSkill — 大而全，但贵且封闭**

奇安信是唯一把"扫描 + 安全市场 + 企业方案"做成闭环的。五维检测（LLM + 静态 + 沙箱 + 子文件 + 外链情报）是检测维度最全的。GPT-5.2 做意图一致性量化评分是很好的产品设计。safeskill.cn 的 Skill Hub 思路对了 — 用户不只要扫描，还要一个"可信的 Skill 来源"。

弱点：闭源 + 高价（17,800 元起），开发者社区渗透力弱。两个域名（qianxin.com vs .cn）品牌混乱。实测扫描耗时 82 秒，比宣称的 30 秒慢很多。

**Cisco Skill Scanner — 技术最硬，但太学术**

CFG + 前向数据流分析（fixpoint 算法）是所有竞品中静态分析能力最强的。能真正追踪数据从 source（凭证文件）流向 sink（网络发送），不是简单的模式匹配。插件架构和 SARIF 输出说明团队有扎实的工程功底。

弱点：太"工具化"，没有产品包装。仅支持 Python AST 分析，不覆盖 shell/JS。没有 LLM 语义分析，面对混淆手段（Base64 + 变量拼接）可能漏报。没有 Web 界面、没有商业化、社区驱动。

**SkillSafe — 产品化最好，但扫描太浅**

SkillSafe 是唯一把 Skill 安全当作"注册中心"来做的。双向加密验证（发布者扫 + 消费者再扫 + SHA-256 Tree Hash）是一个简洁有效的安全模型 — 不追求检测一切恶意，而是确保"你安装的和发布者上传的一模一样"。产品细节打磨好：版本控制、yank 能力、Badge API、默认私有分享、零依赖 CLI。定价亲民（$9/月），"安全不应该收费"的理念讨好开发者。

弱点：扫描引擎太弱。仅检测三类威胁（恶意模式、凭证窃取、Prompt 注入），没有 LLM 语义分析，没有沙箱，没有威胁情报。本质上是一个**带安全验证的 npm registry**，不是一个安全扫描器。

**Socket — 数据最多，但不独立**

Socket 最大的优势是 60K 规模和跨语言覆盖（10+ 语言）。供应链安全的经验从 npm/PyPI 迁移过来，对 typosquatting、维护者劫持等攻击理解深刻。94.5% 检测精度是所有竞品中唯一公开精度数据的。

弱点：没有独立面向终端用户的产品，完全依附 Vercel skills.sh。没有 CLI，没有 API，开发者无法直接使用。更像是一个 B2B 能力而不是 B2D 产品。

## 二、可学习的点

### 从 Snyk 学

| 学什么 | 为什么 | ClawSafe 怎么用 |
|--------|--------|----------------|
| **Toxic Flow 检测** | 不只是找恶意代码，而是分析"无害的 prompt 如何触发恶意行为" | SKILL.md 中加入对 prompt → action 链路的分析指引 |
| **自动发现本地 Agent** | 用户不需要指定路径，扫描器自动找到所有 Agent 配置 | CLI 层面加入 `clawsafety discover` 命令 |
| **三种形态并行** | CLI（开发者）+ Web（一次性扫描）+ 后台（企业） | 近期做 CLI + Web，远期做后台 |
| **生态合作** | 与 Vercel/Tessl 合作嵌入注册中心 | 考虑与国内 Skill 平台合作 |

### 从奇安信学

| 学什么 | 为什么 | ClawSafe 怎么用 |
|--------|--------|----------------|
| **意图一致性量化** | "15% 意图一致性（阈值 80%）" 比 "有风险" 三个字信息量大得多 | 在报告中加入意图一致性百分比，作为辅助指标（不替代三档判定） |
| **扫描时间线** | 分阶段展示扫描进度，用户知道"现在在干什么" | 报告中加入时间线，展示每个 Phase 的耗时和发现数 |
| **五维检测 + 综合判定** | 多维度交叉验证比单一维度可靠 | 我们已有类似思路（规则 + LLM + 情报），但可以更结构化展示 |
| **API 设计** | scan / report / search 三端点很简洁 | SaaS 版直接参考这个 API 结构 |

### 从 Cisco 学

| 学什么 | 为什么 | ClawSafe 怎么用 |
|--------|--------|----------------|
| **SARIF 输出** | CI/CD 标准格式，GitHub Code Scanning 直接接入 | CLI 加 `--format sarif` 输出 |
| **Pre-commit Hook** | 在 skill 提交到仓库前自动扫描，左移安全 | 提供 `.pre-commit-hooks.yaml` |
| **CFG 数据流分析** | 追踪数据从 source 到 sink 的完整路径 | LLM 分析中加入数据流追踪提示，让 Claude 画出数据流图 |
| **插件架构** | 允许社区贡献自定义分析器 | 远期考虑，MVP 不需要 |

### 从 SkillSafe 学

| 学什么 | 为什么 | ClawSafe 怎么用 |
|--------|--------|----------------|
| **双向验证模型** | 简单而有效 — 不追求检测一切，而是确保完整性 | 在 CLI 中加入哈希验证：`clawsafety verify <path> --hash <expected>` |
| **零依赖 CLI** | 单文件 Python，用户体验极好 | 我们的 Rust CLI 编译后也是单文件，更好 |
| **Badge API** | 在 GitHub README 中展示安全状态 | SaaS 版提供 Badge API |
| **yank 能力** | 标记某个版本为不安全，下游自动收到警告 | 未来 SaaS 版可以做 |
| **"安全不收费"** | 免费版包含完整扫描，只对存储和 API 限额收费 | 定价策略参考 |

### 从 Socket 学

| 学什么 | 为什么 | ClawSafe 怎么用 |
|--------|--------|----------------|
| **跨语言扫描** | Skill 代码可能是 Python + Shell + JS 混合 | 我们的 LLM 天然跨语言，这是优势 |
| **精度指标公开** | 94.5% 是目前唯一公开精度数据的 | 建立测试集，公开我们的精度数据 |
| **供应链视角** | 不只看单个 Skill，看整个依赖链 | Phase 5 供应链风险分析中加强 |

## 三、竞争格局判断

### 赛道分化为三个层次

```
第一层：平台级（扫描 + Hub + 企业方案）
  → 奇安信 SafeSkill（国内唯一）
  → Snyk Agent Scan（全球标杆）

第二层：工具级（扫描器 + CI/CD）
  → Cisco Skill Scanner（开源最强）
  → Socket（规模最大）
  → SkillSafe（产品化最好）

第三层：嵌入式（Agent 内部扫描）
  → ClawSafe（目前唯一）
```

### ClawSafe 的战略位置

我们不在第一层（没有资源做平台），也不该在第二层（跟 Snyk/Cisco 拼工具没有优势）。

**我们在第三层 — "嵌入式安全"**，这是一个没人做的位置：

- Snyk 的 MCP Server 模式有点接近，但本质还是外部工具
- ClawSafe 是 **Agent 自己的安全能力**，不是外挂
- 分发优势：不需要 `pip install`、不需要 `cargo install`，放到 skills 目录就能用
- 体验优势：用户用自然语言说"扫描这个 skill"，不需要学命令行

### 风险

1. **第三层太小了** — 如果所有人都去用第一层/第二层的工具，嵌入式这条路没市场
2. **依赖 Agent 平台** — 如果 OpenClaw 不起来，或者 Agent 平台自己内置了扫描功能，我们就没了
3. **LLM 成本** — 每次扫描消耗 Agent 的 context，大规模使用有成本问题

### 应对

- 保持 CLI 作为第二层的存在，不放弃传统工具形态
- Skill 做体验和深度，CLI 做 CI/CD 和自动化
- 两条腿走路，不赌单一形态

## 五、一句话总结

**Snyk 做得最好，奇安信做得最全，Cisco 技术最硬，SkillSafe 产品化最好，Socket 数据最多。ClawSafe 的机会在"嵌入式安全"这个无人区 — 让 Agent 自己保护自己。**
