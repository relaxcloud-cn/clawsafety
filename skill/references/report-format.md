# ClawSafe 报告格式

## 结构

报告由 5 个部分组成，Markdown 格式输出。

---

### 第 1 部分 — 头部

```markdown
# ClawSafe 安全审计报告

| 字段 | 值 |
|------|-----|
| 技能名称 | <name> |
| 路径 | <path> |
| 扫描时间 | <ISO-8601> |
| 扫描器 | ClawSafe v<version> |
```

---

### 第 2 部分 — 结论

三选一：**恶意** / **有风险** / **安全**

```markdown
## 结论：恶意

该技能包含已知 C2 IP 地址和数据窃取代码，应立即删除。
```

```markdown
## 结论：有风险

该技能存在 3 项无法被声明用途解释的可疑行为，建议人工审查。
```

```markdown
## 结论：安全

该技能行为与声明用途一致，未发现安全问题。
```

---

### 第 3 部分 — 发现列表

每个发现一个区块。

```markdown
## 发现

### 1. [严重性] 标题

- **是什么**：具体描述这段代码做了什么
- **在哪里**：`file.py:42` — `function_name()`
- **为什么危险**：解释风险原因
- **怎么修**：修复建议
- **情报关联**：Intel Studio 查询结果（如有）
```

如无发现：`> 未发现安全问题。`

---

### 第 4 部分 — 数据流

```markdown
## 数据流

[输入]                    [处理]                    [输出]
用户输入              -->  LLM 调用              -->  文本响应
环境变量 (API_KEY)    -->  HTTP 请求             -->  JSON 结果
本地文件 (~/.ssh)     -->  读取 + base64 编码    -->  POST 外发  ← 恶意
```

标记所有将敏感数据发往外部的流向。

---

### 第 5 部分 — 处置建议

```markdown
## 处置建议

1. 立即删除该技能（`rm -rf <path>`）
2. 检查是否已有数据泄露（审查网络日志）
3. 轮换可能已泄露的凭证（SSH 密钥、API Key）
```

对于"安全"结论的技能，此部分可省略或写改进建议。
